# Как создать заказ

{% note info %}

**Заказ** — это формализованный документ в системе, описывающий конкретный объем работ и услуг, подлежащих исполнению.

{% endnote %}

Создать заказ возможно двумя способами:
* из подсистемы «Создание заказа» — в этом случае начинайте с шага 1 «выбор заказчика»;
* из карточки поручения по договору — информация о заказчике, договоре и поручении предзаполняется: начинайте с шага 2 «конфигуратор заказа». 

## 1. Заполнение первичной информации

![_screen_request_first_page](_images/_screen_request_first_page.png){.center width=1200}

### Шаг 1. Выбор заказчика

Укажите заказчика, договор и поручение.
Без этих данных создать заказ не получится. 

### Шаг 2. Конфигуратор заказа

Информацию об оборудовании, предоставляемых услугах и адресной программе можно загрузить с использованием excel-шаблона (вкладка «загрузка файла») или вручную (вкладка «мануальный режим»).

{% list tabs %}

- Загрузка файла

  {% note tip %}
        
  Рекомендуется использовать шаблон, если в адресной программе более 10 пространств. 
  Для небольших заказов используйте мануальный режим.  
        
  {% endnote %}

  **Заполнение и загрузка шаблона**
    1. По команде «Шаблон» скачайте excel-файл. 
    2. Заполните информацию, следуя текстовым подсказкам в файле. Обязательные поля помечены желтым цветом. 
    3. Если требуются дополнительные поля, добавьте нужное количество столбцов до разделителя в системе. **Наименование столбца обязательно**, иначе файл не будет прочитан.
    4. Загрузите заполненный шаблон по команде «Прикрепить файл» или перетащите в область загрузки. Файл загрузить можно только 1. 
    ![_gif_request_1](_images/_gif_request_1.gif){.center width=1200}

  **Проверка загруженной информации и конкретизация**
    После загрузки необходимо проверить правильность сопоставления данных из шаблона с данными из системы.

    - **Пространство списания** 
    Несопоставленное пространство будет окрашено в красный, а вверху страницы появится уведомление.
    Строка сопоставленного пространства будет выделена зеленым и отмечена галочкой. 
    
    Для несопоставленного пространства необходимо подобрать адрес, а для сопоставленного пространства - можно поменять: выбрать из выпадающего списка существующий или создать новый по команде «+». 
 
    ![_screen_request_spisanie_03](_images/_screen_request_spisanie_03.png){.center width=600}

    При создании нового пространства адрес и код пространства заказчика проставляются из загруженного шаблона, оставшиеся поля заполняются вручную.  

    - **Сопоставление номенклатуры**
    Из справочника в системе подбирается номенклатура, из которой оно состоит, и указывается требуемое количество для комплектации одного оборудования. 
    
    Если сопоставление не удалось, то необходимо заменить или добавить номенклатуру из справочника/создать новую из выпадающего списка по команде «+». 

    ![_screen_request_nomenc_05](_images/_screen_request_nomenc_05.png){.center width=1200}

  - **Даты поставок**
    Укажите плановые даты поступления номенклатуры. 
    Можно указать несколько дат поставок, если номенклатура поставляется партиями. 
    ![_screen_request_date_supply](_images/_screen_request_date_supply.png){.center width=1200}

  - **Сопоставление услуг**
    На данном шаге сопоставляются услуги. 

    Если сопоставление не удалось, то можно выбрать услугу из предопределенного выпадающего списка. **Добавить новую услугу нельзя**. 

    ![_screen_request_new_service_07](_images/_screen_request_new_service_07.png){.center width=1200}

- Мануальный режим

  **Настройка шаблона**
  Первый шаг — настройка полей для задачи. 
  * Добавьте все необходимые поля в разделе «Дополнительные поля». В карточке задачи поля будут отображаться в том порядке, в котором они добавлены. Порядок можно поменять, перетащив мышкой требуемое поле на нужное порядковое место.
  * Если изменения не требуется или все поля уже добавлены, нажмите команду «Сохранить». После сохранения шаблон поменять нельзя.
  
  **Добавление задач по заказу**
  1. Начните создание заказа с внесения задач по команде «Добавить». 
  ![_screen_request_space_11](_images/_screen_request_space_11.png){.center width=1200}

  1. Укажите пространства списания и поступления оборудования по команде «+». Можно выбрать пространство из уже внесенных в справочник или создать новое. 
  ![_screen_request_space_12](_images/_screen_request_space_12.png){.center width=1200}

  1. Добавьте все оборудование, с которым будут работать в заказе, указав:
  - наименование оборудования;
  - наименования номенклатуры и то количество, которое необходимо для сборки 1 оборудования; 
  - услуги, которые оказываются по 1 оборудованию, и их количество.

  Когда вся информация по задаче внесена воспользуйтесь командой «Добавить».
  ![_screen_request_space_13](_images/_screen_request_space_13.png){.center width=1200}

  В заказе может быть несколько задач, которые можно удалить или отредактировать. 

{% endlist %}

### Шаг 3. Техническое задание

Опишите все требования, необходимые для выполнения работ по заказу.
Можно прикрепить до 21 файла до 5 МБ для конкретизации ТЗ.

![_screen_request_TZ_08](_images/_screen_request_TZ_08.png){.center width=1200}

### Шаг 4. Даты начала и завершения заказа и назначение исполнителя

Для заказа указывается:
- планируемая дата начала — выставляется автоматически на основании первой даты поступления номенклатуры, при необходимости можно изменить вручную; 
- планируемая дата завершения — может быть фиксированной (выбирается вручную) или плавающей (формируется автоматически на основании выбранных операций и расчета);
- ответственный менеджер — контрагент, который будет отвечать за расчёт.

![_screen_request_manager_09](_images/_screen_request_manager_09.png){.center width=1200}

Далее, выводится окно со всей информацией, внесенной по заказу.
После команды «Закрыть» менеджер, назначенный ответственным по данному заказу, увидит новую задачу в списке мероприятий.

![_screen_request_end_10](_images/_screen_request_end_10.png){.center width=1200}
